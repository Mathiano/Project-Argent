# Project Argent — Working Agreement (how this team operates)

Orientation doc for Claude Code (CC). Read once; it governs how sprints run. This is process, not design — design lives in the other docs. If process and a kickoff ever conflict, the kickoff wins for that sprint, but flag the conflict.

## The three roles

- **Mathias — Creative Director.** Owns the vision and every taste call: design decisions, art approval, what's fun. Plays every build. Final veto on everything. Relays between the design chat and CC.
- **Design authority (the design chat, "Fable/Claude").** Owns specs, boss cards, balance rulings, the type chart, content commissioning, sprint kickoffs, and now the **systematic visual layer** (tilesets, UI polish, scene composition — delivered as data/assets). Reads CC's audits. Sizes refactors. Does NOT write repo code.
- **CC — Implementation.** Owns the codebase: engine, game layer, sim, tests, CI. Builds against the docs and kickoffs, gated by sims. Flags ambiguities instead of inventing design. Writes its own kickoff files from pasted briefs.

## Division of labor on art (important — recently changed)

- **Mon/creature sprites:** generated by Gemini, approved one at a time by Mathias, ingested via `tools/sprite_ingest.py`. CC does not produce creature art.
- **Tilesets, UI, scene composition, the systematic visual layer:** authored by the design chat as data/assets, approved by Mathias, ingested by CC into a data-driven loader. CC does not hand-design this art — it builds the *system* that consumes it and ships a clearly-marked placeholder set so the pipeline is end-to-end. Real assets drop in as data, no code change.
- **Everything visual passes Mathias's eye before it becomes canon.**

## The two gates (non-negotiable)

1. **The sim gate.** Any change to combat numbers, AI, or a boss kit re-runs the relevant ladder before merge. Ladders are vitest regressions, seeded, tolerance-banded, n≥2000. The fixture rival ladder and locked boss ladders must stay **bit-identical** across refactors unless a ruling explicitly re-baselines them — if a refactor moves them, STOP and report the diff, never silently re-lock.
2. **The feel gate (new).** The project pivoted feel-first: a build must be *good to play*, not just correct. Slices end with Mathias playing for feel. The standard is "the first five minutes must delight." Correctness is necessary, not sufficient.

## Operating principles

- **Docs win over code.** When they conflict, flag it; never silently change a design number. Keep specs synced to code (the v0.3.4 sync set the precedent).
- **Flag, don't invent.** Ambiguity or a missing design decision → stop and surface it as a question, with options if helpful. Never resolve a design gap by guessing. CC has done this well; keep doing it.
- **Everything stays editable.** Build data-driven, not hardcoded — tiles, prefabs, maps, dex, moves, type chart, boss cards are all data. We will revise the overworld, the roster, and the balance repeatedly; the architecture must assume that.
- **Push after every task batch.** Green commits never sit unpushed. Repo is PRIVATE until the naming-debt rename pass.
- **Report as audit.** Sprint close = what landed, test count, both ladders' bit-identical status, screenshots, and an explicit "deferred / flagged for design" section. A report with honest seams is a healthy report; a report claiming total success with no seams invites a probe.
- **Tests are the net before the high-wire act.** Widen coverage before risky refactors (the consolidation sprint before 6v6 was the model).
